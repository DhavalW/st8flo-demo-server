The problem statement




Humans think visually, not in text.


Manually decoding text based code into a visual mental map,
is a time consuming and taxing activity,
but still required by every developer, whether collaborating on a project, or working solo

which leads to gaps in understanding, and therefore bugs + longer turnaround times



The Solution



What if


you never needed to build mental maps from code,

in the first place ?





St8Flo.js

Text is for machines | Visual, is for humans



Your app's flowchart, itself becomes the app !




Understand & design your code's structure visually




Edit the flowchart directly to change the app's behaviour.




No more need to decode and undertand heaps of text

Save hours on development time & therefore costs


Increase team productivity


Reduce turnaround times on projects


& Wow your customers with live troubleshooting

Understand code flow visually, with less errors



Most slowdowns and errors happen when you attempt to create / edit / debug code, without realising that you've missed out on some important architectural details.

St8Flo helps, by keeping things as visual as possible.

Rapid prototyping & build as you brainstorm


Add & Remove blocks representing various functions within your app, as you brainstorm its structure.

Once you're satisfied, simply fill in the code for each block to ready your prototype for execution.

Deploy on device to test (debug & modify if necessary), till your app performs to expectation

Visualise & simplify app state management and design


State management gets tricky in a complex application, becausing keeping a track of multiple states & their impacts to the flow of execution, is hard to do manually.

Fortunately, St8Flo handles that for you automatically, allowing you to focus on getting it right

Collaborate more intuitively across teams


Software is often designed by teams, rather than a single developer.

St8Flo enables you to show, discuss and edit modifications directly on the flowchart, in real time, together with your team... allowing for a more rapid, intuitive collaboration effort.

Isolate and debug errors in a snap, using visual execution traces


See a visual trace of your app's execution on the flowchart, as it happens.

Isolate errors quickly, by identifying the block where it deviates from the expected route.

Error messages are also bubbled up and shown in the debug pane, for faster resolution

One click sync & deployment to client


Pull session to debug, directly from the client's device. Make the corrections & push it back as an update to the client, in a single click.

No need to go through app stores, ( beyond the initial install )

Troubleshoot client complaints, LIVE and provide immediate resolutions


St8Flo allows you to directly debug the client, from wherever you are, even when in production & in use by the client

Imagine debugging & providing a resolution, even as the client describes his problem over the phone !

Other key Features

Use with ANY Javascript framework, in the browser or server, or even standalone mobile apps


St8Flo is designed to be as generic as possible,
and can be used with technologies like angular & node.js alike... even with hybrid mobile app platforms like Ionic



Build entirely new apps / websites
OR integrate into your existing ones


St8Flo apps can be packed into independent modules, which can be integrated into your existing javascript software through a simple "require()"



Reduce spaghetti code & code bloat


Take advantage of the inherent DRY design, that emerges as a result of seperating flow-control logic from functional logic

Optimize for performance / memory, by simply swapping St8Flo runtimes


Build distributed computing enabled apps


Being modular, app execution can be split across multiple devices / browsers on a per block basis, without any additional modifications to the code

Build modules for repeated use


Package frequently required functionality (Eg REST interfaces, Facebook Login APIs) as modules that can be included into other St8Flo apps.





Use functional / declarative coding, instead of the Visual IDE




Team collaboration features, with roles & permissions
