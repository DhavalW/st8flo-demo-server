So, you just bull-dozed through 10,000 lines of code


which is an endless stream of text about the size of an average English novel,
reading which is as enjoyable, as having to pluck all your hair out, one by one...

possibly filled with sloppy formatting, terrible indentation,
& impossibly cryptic or verbose naming conventions

passed on to you by devs, who almost fanatically believed
that hard-to-read code was the new black





Then wasted countless hours,
constructing a mental flowchart of it,


(& probably several more, just beating the sheer fatigue that set in )

while deconstructing that 10k line code, line-by-line, function-by-function,

even in the face of a tight deadline,





All, so that you understand & mentally map the code flow


& find the cause of that ONE, retarded, but critical bug,
hidden deep within the code somewhere.





Only to be suddenly interrupted,


because < x > person wanted < y > done immediately



& NOW you have to repeat it all, all over again !


additionally also for the, some 100 odd, remaining bugs...


Sound familiar ?


frustratingly perhaps ?

enough to maybe try a better approach ?
